✏️ Before You Upload — Edit Two Lines
Open index.html in any text editor (Notepad, TextEdit, VS Code) and find these two lines near the bottom of the file. Change them to your actual GitHub username and repo name.
✓
01
Edit index.html — set your GitHub username and repo name
Required

Open index.html in a text editor. Press Ctrl+F (or Cmd+F on Mac) and search for CHANGE THIS. You'll find two lines:

const GITHUB_USER = 'your-username';   // ← CHANGE THIS
const GITHUB_REPO = 'mission-site';    // ← CHANGE THIS

Change 'your-username' to your actual GitHub username, and 'mission-site' to whatever you name your repository. For example:

const GITHUB_USER = 'mikegledhill';
const GITHUB_REPO = 'mission-site';

Save the file. That's the only code edit you ever need to make.

🐙 GitHub Setup
Create a repository and enable GitHub Pages. This is where your site will live — for free, forever.
✓
02
Create a GitHub account (if you don't have one)
One-time

Go to github.com and sign up for a free account. Use a family email address you check regularly.

Your GitHub username will be part of your site URL — e.g. mikegledhill.github.io/mission-site — so pick something clean and easy to share.

✓
03
Create a new GitHub repository
Required
  1. Go to github.com/new
  2. Repository name: mission-site (or any name — just match it in step 1)
  3. Set to Public (required for free GitHub Pages)
  4. Leave everything else unchecked
  5. Click Create repository
💡 The repo name becomes part of your URL: username.github.io/repo-name
✓
04
Upload all 7 files to the repository
Required
  1. In your new repo, click Add file → Upload files
  2. Drag and drop all 7 files into the upload area:
    index.html, admin.html, data.json, .nojekyll, _config.yml, 404.html, README.md
  3. At the bottom, write a commit message: Initial upload
  4. Click Commit changes
⚠️ Make sure .nojekyll is included — it's a hidden file (starts with a dot). On Mac, press Cmd+Shift+. to show hidden files before dragging.
✓
05
Enable GitHub Pages
Required
  1. In your repo, click Settings (top tab)
  2. In the left sidebar, click Pages
  3. Under Source, select Deploy from a branch
  4. Choose branch: main, folder: / (root)
  5. Click Save

Wait 1–3 minutes. Your site will be live at:

https://YOUR-USERNAME.github.io/mission-site/
💡 GitHub will show a green checkmark under Settings → Pages when it's live. You can also check the Actions tab to see the deployment progress.
🔐 Admin Panel Setup
Set up the admin panel so you can edit and publish changes directly from the browser.
✓
06
Get a GitHub Personal Access Token (PAT)
One-time

This token lets the admin panel push changes directly to GitHub from your browser. It's stored only in your browser — never in any file on the internet.

  1. Go to: github.com/settings/tokens
  2. Click Generate new token → Generate new token (classic)
  3. Note/name: Mission Site Admin
  4. Expiration: No expiration (or 1 year — you'll need to renew it)
  5. Scopes: check repo (the top checkbox under repo)
  6. Scroll down → click Generate token
  7. Copy the token immediately — GitHub won't show it again!
⚠️ Treat your PAT like a password. Don't share it or put it in any file. It is stored only in your browser's localStorage.
Your token will look like: ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxx
✓
07
Configure the Admin Panel with GitHub credentials
One-time
  1. Visit your live site: https://YOUR-USERNAME.github.io/mission-site/admin.html
  2. Log in with the default password: mission2024
  3. In the left sidebar, click 🐙 GitHub / Publish
  4. Fill in:
    • GitHub Username: your username
    • Repository Name: mission-site
    • Branch: main
    • Personal Access Token: paste your token from step 6
  5. Click Save GitHub Config

The status indicator at the top should turn green: 🟢 username/mission-site

✓
08
Change the admin password
Required

The default password is mission2024. Change it before sharing the site.

  1. In the admin panel, go to ⚙️ Settings
  2. Under Change Admin Password:
  3. Current password: mission2024
  4. Enter and confirm your new password
  5. Click Update Password
💡 The password is stored in your browser's localStorage. If you clear your browser data, it resets to mission2024. Write it down somewhere safe!
🗺️ Final Customization
✓
09
Draw your mission boundary on the map
Required
  1. In the admin panel, go to 🗺️ Mission Map Editor
  2. Click the polygon tool (pentagon icon in the left toolbar)
  3. Click point-by-point around the edge of your mission territory
  4. Double-click to close the shape
  5. When prompted "Use as mission boundary?" — click OK
  6. Click Save All Map Data
  7. Click 🐙 Push to GitHub

Alternatively, if you have a KML or GeoJSON file of the mission boundary, paste the coordinates array in the Import GeoJSON box.

✓
10
Do a test publish — push your first real update
Required
  1. In the admin panel, go to 📝 Weekly Updates
  2. Write your first update (or use the placeholder Week 47)
  3. Click 🚀 Publish Update
  4. Click 🐙 Push to GitHub in the top toolbar
  5. Enter a commit message like Add week 47 update
  6. Watch the green status indicator confirm success
  7. Wait 1–2 minutes, then visit your live site to confirm it updated
💡 After this, every Monday you just: write the update → click Publish → click Push to GitHub. That's it!

Set up a custom domain (e.g. eldergledhill.com)
Optional

If you want a custom domain instead of username.github.io/mission-site:

  1. Buy a domain from Namecheap, GoDaddy, or Google Domains (~$12/year)
  2. In your domain registrar, add these DNS records:
Type: A      Name: @    Value: 185.199.108.153
Type: A      Name: @    Value: 185.199.109.153
Type: A      Name: @    Value: 185.199.110.153
Type: A      Name: @    Value: 185.199.111.153
Type: CNAME  Name: www  Value: YOUR-USERNAME.github.io
  1. In GitHub repo → Settings → Pages → Custom domain → enter your domain
  2. Check "Enforce HTTPS"
